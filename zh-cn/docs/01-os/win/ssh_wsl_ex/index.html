    ssh连接wsl后执行Windows程序

    2023/01/011 分钟阅读 Windows 开发环境 开发环境WindowsWsl

    目录

    1. 准备工作

    wsl需要安装openssh-server，有可能需要修改/etc/sshd_config中端口号。

    2. wsl运行程序

    ssh连接wsl后，需要将Windows中PATH环境变量指定的路径添加到wsl的PATH变量中。格式如下：

     1export PATH="${PATH}:/mnt/c/Program Files/Microsoft SQL Server/120/Tools/Binn/"
 2export PATH="${PATH}:/mnt/c/Program Files/dotnet/"
 3export PATH="${PATH}:/mnt/c/Programs/Git/cmd"
 4export PATH="${PATH}:/mnt/c/Programs/Graphviz/bin"
 5export PATH="${PATH}:/mnt/c/Programs/Meld/"
 6export PATH="${PATH}:/mnt/c/Programs/Microsoft VS Code/bin"
 7export PATH="${PATH}:/mnt/c/Programs/Python/Python310/"
 8export PATH="${PATH}:/mnt/c/Programs/Python/Python310/Scripts/"
 9export PATH="${PATH}:/mnt/c/Programs/VMware/VMware Player/bin/"
10export PATH="${PATH}:/mnt/c/Programs/VanDyke Software/Clients/"
11export PATH="${PATH}:/mnt/c/Programs/swigwin-4.1.1"
12export PATH="${PATH}:/mnt/c/Users/WDN/.dotnet/tools'"
13export PATH="${PATH}:/mnt/c/Users/WDN/AppData/Local/Microsoft/WindowsApps"
14export PATH="${PATH}:/mnt/c/WINDOWS"
15export PATH="${PATH}:/mnt/c/WINDOWS/System32/OpenSSH/"
16export PATH="${PATH}:/mnt/c/WINDOWS/System32/Wbem"
17export PATH="${PATH}:/mnt/c/WINDOWS/System32/WindowsPowerShell/v1.0/"
18export PATH="${PATH}:/mnt/c/WINDOWS/system32"
19export PATH="${PATH}:/mnt/c/Windows"
20export PATH="${PATH}:/mnt/c/Windows/System32/OpenSSH/"
21export PATH="${PATH}:/mnt/c/Windows/System32/Wbem"
22export PATH="${PATH}:/mnt/c/Windows/System32/WindowsPowerShell/v1.0/"
23export PATH="${PATH}:/mnt/c/Windows/system32"

    添加环境变量后，可以开始调用Windows程序，如powershell.exe。

    如果上述环境变量有遗漏，可以使用powershell来获取后继续添加。

    3. powershell查看环境变量

    在 ‌PowerShell 中，可以通过使用 Get-ChildItem 命令配合环境变量名称来查看环境变量的值。例如，要查看 PATH 环境变量的值，可以执行以下命令：

    1Get-ChildItem -Path env:PATH
2echo ${env:Path}

    这将列出 PATH 环境变量中包含的所有路径。类似地，要查看其他环境变量，只需将 PATH 替换为相应的环境变量名称即可。例如，查看 SystemRoot 环境变量：

    1Get-ChildItem -Path env:SystemRoot

    此外，如果需要查看所有环境变量的列表，可以使用 env: 驱动器直接列出其下的所有项：

    1Get-ChildItem env:*

    这将显示所有环境变量的名称和值。这些命令在 PowerShell 中直接执行，无需额外的设置或配置。‌
